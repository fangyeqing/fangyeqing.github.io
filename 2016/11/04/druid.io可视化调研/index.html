Druid可视化

在Druid.io可视化问题上，调研了几个druid官方推荐工具，都自己试验了一遍，最后选择superset。

name 描述 实现 其他
grafana grafana的druid插件，比较简陋，github一年不更新了 js
Metabase 支持数据库种类多，启动方便，支持json查询。图形化查询，只能有一个聚合字段，两个维度 Clojure
imply-pivot 基于Plywood，部署方便，能构造复杂的查询。目前已经闭源了，没法二次开发 js
airbnb/superset 权限管理完善，图形可定制性也比较高，github持续更新,集合了metabase的Dashboard和pivot的查询可定制性优点，部署相对麻烦 python+js

metabase使用

需要jdk1.6以上的环境，使用的jdk1.8

然后就可以访问：
http://localhost:3000/

初始化步骤：

  • 填写个人基本信息和用户名密码
  • 连接数据库基本信息，数据库类型选druid，取数据库名字，填host和broker port。
  • 关闭收集数据选项（Allow Metabase to anonymously collect usage events）

    使用

  • 新建dashboards，可以把每个表新建一个dashboard
  • 新建question，每个question相当于一个查询语句，但是每个question只能选一个聚合字段，可以显示不同类型的图形

imply-pivot使用

环境

node+npm环境

然后就可以访问了：
http://localhost:9090/

使用

  • 左上角选择表名，右上角选择展示方式
  • 左上的Dimensions包括时间和维度，可以拖到filter为where条件，拖到split为group by
  • 查询之后的表结果，可以选择之后点右键，select可以添加查询条件;view raw data，相当于drill down

图中split条件为xxx_id,xxx_id,xxx_id,xx_id。
filter中条件为时间为1天内，网络类型为4G,xxx-id为2。
点击某xxx=221660，xxx_id=1340之后可以添加这些条件，点select为查询添加了这些条件之后的结果，点view raw data为查询添加了这些条件中所有的结果。

顶部选项卡

Dashboard：展示板，可以每张表建一个，展示这张表需要展示的
Slices：Dashboard中的分片，相当于一个查询语句
Source：数据源

Cluster配置

顶部选项卡Source-Druid Cluster-“+”新建

选项 填入内容
Cluster druid集群名，随意
Coordinator Host coordinator地址
Coordinator Port coordinator端口
Coordinator Endpoint 默认值：druid/coordinator/v1/metadata
Broker Host broker地址
Broker Port broker端口
Broker Endpoint 默认值：druid/v2
Cache Timeout 缓存时间：s

DataSource配置

Source-Refresh Druid MetaData

读取元数据，第一次可能会等待几分钟,开调试模式可以看到debug日志里自动同步了druid里的所有表。

日志输出内容：

Source-DataSource

在对应的dataSource处edit record

  • List Druid Column选项卡中，可以看到自动导入了column，但是metrix聚合字段并没有选择聚合方法，编辑选择SUM或者MIN，MAX之类的聚合方法。druid lookup也可以在本选项卡中配置。
  • Detail选项卡中，Time offset填入时区偏移,配置已配成本地时区。点save保存。
  • List Druid Metrix选项卡中，List Druid Column中刚才添加的聚合字段会看到。同时也可以通过“Metric”字段编辑自定义图中显示的中文或者post-agg。

post-agg示例：展示率=展示/竞价

lookup示例：把sponsor_id对应到广告商名字

DashBoard添加

  • 步骤
    以Source-Druid DataSource做入口，选某个表进行编辑，然后保存成Slices，添加到DashBoard
  • 界面
    左边栏为查询条件，右边为查询结果。

Query为查询，Save为保存到Slices
Table View为结果显示样式（图，表类型），
查询出结果之后，左边栏可以对图形属性进行编辑

角色权限配置

Admin新建用户时，有几种权限可以选择Admin、Alpha、Gamma、Public。

  • Admin：有所有权限，创建用户、权限管理、访问所有数据源、
  • Alpha：可以访问、添加、修改所有数据源。添加和修改面板和切面。
  • Gamma：访问dashboard需要添加访问datasource的权限。搜索：“datasource access on”。另外，配置时，可以搜索druid有关的权限，全部删掉。
