编译前端部分

  • npm install:安装依赖到node_modules(-g参数安装到全局，/usr/local)，先检查当前目录下node_modules有没有模块，没有的话会从registry中下载。参考
    • 发出npm install命令
    • npm 向 registry 查询模块压缩包的网址
    • 下载压缩包，存放在~/.npm目录
    • 解压压缩包到当前项目的node_modules目录
  • npm run xxx：执行package.json，xxx可以在”scripts”中找到，build、prod之类的都是自己配的。每当执行npm run，就会自动新建一个 Shell，在这个 Shell 里面执行指定的脚本命令。会生成dist目录

总之，上面两个命令，会生成node_modules和dist。

当然没那么顺利

先卡在78%，然后报错。

办法1

报错是用了两种版本的react，如果只把他缺少的ReactDOM从0.15.3复制过去，又会有新的错误提示少了东西。

先把react的lib删掉，然后把.15.3.2\@react的lib拷贝过去

办法2

这样会缺少前端的组件,改成直接从pip安装好的superset中拷贝下来。static/assets只是一个软连接指向assets

后端部分

Pycharm中导入项目后，可能一直扫描建索引，需把node_modules排除在项目外，assets和static里都有。

在ide中，项目superset目录下添加run.py，然后run就可以在浏览器8088访问到

superset-python安装

配置

如果之前配置过，直接拷贝配置.superset/db也可

顶部选项卡

Dashboard：展示板，可以每张表建一个，展示这张表需要展示的
Slices：Dashboard中的分片，相当于一个查询语句
Source：数据源

Cluster配置

顶部选项卡Source-Druid Cluster-“+”新建

选项 填入内容
Cluster druid集群名，随意
Coordinator Host coordinator地址
Coordinator Port coordinator端口
Coordinator Endpoint 默认值：druid/coordinator/v1/metadata
Broker Host broker地址
Broker Port broker端口
Broker Endpoint 默认值：druid/v2
Cache Timeout 缓存时间：s

DataSource配置

Source-Refresh Druid MetaData

读取元数据，第一次可能会等待几分钟,开调试模式可以看到debug日志里自动同步了druid里的所有表。

日志输出内容：

Source-DataSource

在对应的dataSource处edit record

  • List Druid Column选项卡中，可以看到自动导入了column，但是metrix聚合字段并没有选择聚合方法，编辑选择SUM或者MIN，MAX之类的聚合方法。druid lookup也可以在本选项卡中配置，配置方法在文末。
  • Detail选项卡中，Time offset填入时区偏移,配置已配成本地时区。点save保存。
  • List Druid Metrix选项卡中，List Druid Column中刚才添加的聚合字段会看到。同时也可以通过“Metric”字段编辑自定义图中显示的中文或者post-agg。

DashBoard添加

  • 步骤
    以Source-Druid DataSource做入口，选某个表进行编辑，然后保存成Slices，添加到DashBoard
  • 界面
    左边栏为查询条件，右边为查询结果。

Query为查询，Save为保存到Slices
Table View为结果显示样式（图，表类型），
查询出结果之后，左边栏可以对图形属性进行编辑

角色权限配置

Admin新建用户时，有几种权限可以选择Admin、Alpha、Gamma、Public。

  • Admin：有所有权限，创建用户、权限管理、访问所有数据源、
  • Alpha：可以访问、添加、修改所有数据源。添加和修改面板和切面。
  • Gamma：访问dashboard需要添加访问datasource的权限。搜索：“datasource access on”。另外，配置时，可以搜索druid有关的权限，全部删掉。
